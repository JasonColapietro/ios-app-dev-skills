# /ios-review-notes

Input: `$ARGUMENTS` - app path, release brief, reviewer account, or review issue.

Use the `ios-app-store-release` and `ios-aso-launch` skills.

Write review notes that help Apple test the real app:

- what the app does;
- login or demo account instructions without exposing private credentials in
  tracked files;
- paid feature or subscription behavior;
- device permissions used and why;
- any web shell or remote-content explanation;
- support contact placeholder if needed.

Output safe review notes and any private values that must be filled locally.
