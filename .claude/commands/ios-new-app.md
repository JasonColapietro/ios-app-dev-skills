# /ios-new-app

Input: `$ARGUMENTS` - app name, concept, or existing project path.

Use the `ios-swiftui-product` and `ios-app-factory` skills.

## Requirements

- Prefer native SwiftUI and a testable core package.
- Generate or identify bundle ID, scheme, target, deployment version, and
  package layout.
- Include onboarding/empty state, core flow, history/saved set, settings, and
  review prompt after success.
- Keep demo harnesses DEBUG-only.
- Build or provide exact build command before calling the scaffold done.

Output implementation plan, touched files, build command, and next step.
