# /ios-store-assets

Input: `$ARGUMENTS` - app path, URL, screenshots, keyword, or listing request.

Use the `ios-aso-launch` and `ios-screenshot-taker` skills.

Create the listing asset plan:

- target keyword and positioning;
- screenshot order and captions;
- required simulator states;
- app name, subtitle, keyword field, description, and promotional text;
- support, marketing, and privacy URLs;
- feature-truth check against the actual app.

Output metadata files to create or update plus the screenshot capture command.
