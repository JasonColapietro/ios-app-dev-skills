# /ios-privacy

Input: `$ARGUMENTS` - app path, URL, SDK list, privacy concern, or release brief.

Use the `ios-app-store-release`, `site-to-ios-app`, and `ios-swiftui-product`
skills.

Build the privacy truth table:

- SDKs, analytics, auth, payments, ads, crash reporting, and media permissions;
- App Privacy answers;
- `PrivacyInfo.xcprivacy` needs;
- Info.plist usage strings;
- privacy policy, terms, support, and account deletion URLs;
- reviewer notes for permissions or login.

Output missing links, required disclosures, and release blockers.
